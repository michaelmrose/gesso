#+title: Gesso
#+subtitle: Server-rendered live UI for Clojure
#+options: toc:nil

A small Clojure library for building server-rendered applications with Hiccup, HTMX, SSE, and Biff.

The basic model is ordinary HTTP: handlers receive requests, views return HTML, and HTMX swaps fragments into the page. For live updates, the server sends an SSE invalidation event and the browser asks for the affected fragment again.

Gesso adds helpers for reusable components, live fragments, client-side continuity across swaps, and theme CSS generation.

* Install

Use the released Git coordinate in ~deps.edn~:

#+begin_src clojure
{:deps
{michaelmrose/gesso
{:git/url "https://github.com/michaelmrose/gesso.git"
:git/tag "v0.2.0"
:git/sha "3f22e0c3e3cd34fca9cbb5a03625e203d5b3ad37"}}}
#+end_src

For local development against a sibling checkout:

#+begin_src clojure
{:deps
{michaelmrose/gesso {:local/root "../gesso"}}}
#+end_src

* Try it

The easiest starting point is Gessokit, which contains a deps-new template and an example app.

#+begin_src sh
git clone https://github.com/michaelmrose/gessokit
cd gessokit
bb to-template
bb new ../myapp myapp
cd ../myapp
clojure -M:dev dev
#+end_src

The generated app includes the basic Gesso project shape, theme build setup, routing, HTMX forms, SSE invalidation, live fragments, toasts, and continuity across fragment replacement.

For local framework development, clone Gesso and Gessokit side by side and temporarily point Gessokit at the local checkout. For normal generated projects, use the released Git dependency.


* Live fragments

A live fragment is a server-rendered piece of HTML that can be refreshed when its backing state changes.

#+begin_src text
data changes
-> server invalidates a live scope
-> browser receives an SSE event
-> HTMX requests the fragment again
-> server returns fresh HTML
-> browser swaps it into place
#+end_src

This keeps the application model server-side without giving up live UI.

* Continuity

Replacing a fragment can accidentally lose browser-local state: focus, caret position, scroll anchoring, open cards, or local input state.

Gesso Live can preserve that state across swaps.

#+begin_src clojure
(require '[gesso.live.continuity :as continuity])

(def request-list-continuity
(continuity/preserve
{:scroll {:selector "[data-request-card]"}
:focus true
:boxes [(continuity/details-open
{:selector "details[data-request-card][data-accordion-value]"
:key-attr "data-accordion-value"
:single? true})]}))
#+end_src

That example preserves scroll position, focus/caret state, and the currently open native ~<details>~ card.

* Themes

Gesso can build theme CSS from framework and application component roots.

#+begin_src clojure
((requiring-resolve 'gesso.build.themes/build!)
{:input-dir "resources/themes"
:output-file "resources/public/gesso/app-themes.css"
:component-css-dirs ["path/to/gesso/components"
"src/myapp/components"
"src/myapp/site/components"]})
#+end_src

* Development

Run tests:

#+begin_src sh
clojure -M:test
#+end_src

Create a release tag:

#+begin_src sh
git tag -a v0.2.0 -m "Gesso 0.2.0"
git push origin v0.2.0
#+end_src

#+begin_src sh
git rev-parse 'v0.2.0^{commit}'
#+end_src
