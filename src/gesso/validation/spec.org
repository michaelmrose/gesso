#+TITLE: Gesso Validation Engine Technical Specification
#+DESCRIPTION: The pure-logic bridge translating Malli AST into HTMX/Hyperscript UI states.

* gesso.validation.malli
** (extract-constraints [schema field-kw])
- Input ::
  - =schema=: A compiled Malli schema or raw vector (e.g., =[:map [:email [:string {:min 5 :gesso.error/min "Too short"}]]]=).
  - =field-kw=: Keyword targeting the specific field (e.g., =:email=).
- Returns ::
  #+BEGIN_SRC clojure
  {:rules {:minlength int? :maxlength int? :pattern string? :required boolean?}
   :messages {:min str? :max str? :pattern str? :required str?}}
  #+END_SRC
- Logic ::
  1. Uses =malli.core/ast= or =malli.util= to navigate to the =field-kw= child.
  2. Maps internal Malli keys (=:min=, =:re=) to standard HTML5 attribute keys (=:minlength=, =:pattern=).
  3. Extracts custom error strings looking for =:gesso.error/<type>=, falling back to standard =:error/message=, then library defaults.

** (resolve-error-message [constraints validity-state])
- Input :: The =:messages= map and a string representing the HTML5 ValidityState failure (e.g., "valueMissing", "tooShort").
- Returns :: The resolved string message.

* gesso.validation.scripts
** (gatekeeper-script [messages-map err-id])
- Input :: =:messages= map from =extract-constraints= and the target =err-id= string.
- Returns :: String (Hyperscript).
- Output Example ::
  #+BEGIN_SRC text
  on input or blur
    if my.checkValidity() is false
      if my.validity.valueMissing set #<err-id>.innerHTML to '<messages-map.required>'
      else if my.validity.tooShort set #<err-id>.innerHTML to '<messages-map.minlength>'
      remove .hidden from #<err-id>
      add .aria-invalid='true' to me
    else
      add .hidden to #<err-id>
      set .aria-invalid='false' on me
      trigger validateField
  #+END_SRC
- Logic :: Defers the actual math (regex/length) to the browser's native C++ engine, only waking up Hyperscript to map the failure to your custom Malli string and manage CSS.

* gesso.validation.htmx
** (render-oob-errors [explain-data])
- Input :: Malli =explain-data= map resulting from a failed backend =m/explain=.
- Returns :: Hiccup Vector.
  #+BEGIN_SRC clojure
  [:<>
   [:div {:id "email-error" :hx-swap-oob "innerHTML"} "Email already taken"]
   [:div {:id "password-error" :hx-swap-oob "innerHTML"} "Password too weak"]]
  #+END_SRC
- Logic :: Transforms the path vector in =explain-data= into the stable =id= format matching =gesso.components.field.attr/derive-ids=.

* gesso.validation.core
** (field-plan [schema field-kw err-id])
- Input :: =schema=, =field-kw=, =err-id=.
- Returns :: Map to be injected into the UI component.
  #+BEGIN_SRC clojure
  {:attrs {:minlength 5 :required true}
   :script "on input or blur..."}
  #+END_SRC
- Logic :: The orchestrator. Calls =malli/extract-constraints=, plucks the =:rules= for =:attrs=, and passes the =:messages= to =scripts/gatekeeper-script= to bundle the plan.
